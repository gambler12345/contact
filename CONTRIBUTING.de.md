# Beitragsrichtlinien

Vielen Dank für Ihr Interesse, zu diesem Projekt beizutragen! Wir begrüßen alle Beiträge, ob Fehlermeldungen, Feature-Anfragen, Verbesserungen der Dokumentation oder Code-Änderungen.

## Wie man beiträgt

### Fehler melden

Bitte prüfen Sie vor dem Einreichen eines Fehlerberichts, ob das Problem bereits im [Issue-Tracker](../../issues) gemeldet wurde.

Beim Einreichen eines Fehlerberichts geben Sie bitte folgendes an:
- Einen klaren und beschreibenden Titel
- Schritte zur Reproduktion des Problems
- Erwartetes Verhalten im Vergleich zum tatsächlichen Verhalten
- Alle relevanten Logs, Screenshots oder Fehlermeldungen
- Details zu Ihrer Umgebung (Betriebssystem, Browser, Version usw.)

### Features vorschlagen

Feature-Anfragen sind willkommen! Bitte öffnen Sie ein Issue und fügen Sie folgendes hinzu:
- Eine klare Beschreibung des Features
- Die Motivation oder den Anwendungsfall dahinter
- Alle von Ihnen in Betracht gezogenen Alternativen

### Pull Requests einreichen

1. Forken Sie das Repository und erstellen Sie Ihren Branch von `main`.
2. Nehmen Sie Ihre Änderungen mit klaren, beschreibenden Commit-Nachrichten vor.
3. Stellen Sie sicher, dass Ihre Änderungen die bestehende Funktionalität nicht beeinträchtigen.
4. Öffnen Sie einen Pull Request mit einem klaren Titel und einer Beschreibung Ihrer Änderungen.
5. Verknüpfen Sie alle verwandten Issues mit Schlüsselwörtern (z. B. `Closes #123`).

## Verhaltenskodex

Bitte beachten Sie, dass dieses Projekt mit einem [Verhaltenskodex für Mitwirkende](CODE_OF_CONDUCT.de.md) veröffentlicht wird. Mit der Teilnahme an diesem Projekt erklären Sie sich damit einverstanden, dessen Bedingungen einzuhalten.

## Fragen

Wenn Sie Fragen haben oder Hilfe benötigen, öffnen Sie bitte ein Issue oder lesen Sie [SUPPORT.de.md](SUPPORT.de.md).
